# Звіт про виконання master-plan.md

> Сесія Claude Code Web, 2026-07-18. Гілка `claude/master-plan-docs-v7fnam`,
> база — `main` @ 4.1.1 (`d00f46a`). Підсумкова версія: **4.2.0** (протокол 4.2).

## Важливе обмеження цієї сесії

У середовищі виконання **не було Godot-бінарника** і мережева політика не
дозволила його завантажити (доступні лише пакетні реєстри). Тому:

- Правило №1 плану («тест-драйв після кожного етапу») виконати **не вдалося**.
- Замість цього кожен змінений/новий файл перевірявся `gdparse` (gdtoolkit 4.5,
  синтаксис GDScript 2), а кожне перенесення коду верифіковане скриптами
  «токен-ідентичності»: нормалізовані тіла функцій до/після збігаються
  (з поправкою лише на навмисні `game.`/`hud.`/`WgX.`-кваліфікації).
- **Перед мержем обов'язково прогнати повний тест-драйв:**
  ```
  godot --headless --path . --import
  godot --headless --path . -- --test
  godot --headless --path . -- --test-story
  godot --headless --path . -- --mphost   # + --mpjoin у другому терміналі
  ```
  Плюс Godot згенерує `.uid` для нових скриптів при першому імпорті —
  закомітити їх окремим комітом.

### Виправлення знахідок веб-сесії

- `hud_dialog.gd:252` — компілятор Godot падав (`Cannot infer the type of "t"`);
  gdparse цього не ловив. Фікс: явний тип `Tween`.
- i18n-guard хибно фейлив на легітимно однакових перекладах («Бомба»/«ЛКМ»
  uk==ru). Фікс: перевірка наявності ключа в каталозі локалі
  (`get_translation_object().get_message()`), а не `tr(x)==x`.
- Тестові сейви (`test_*`) переживали запуск: арбалет, вдягнений наприкінці
  `--test-story`, протікав у наступний прогін і ламав items-тести. Фікс:
  `_reset_test_saves()` на старті харнеса (`--continue` не чіпає).
- `.uid` нових скриптів згенеровані й закомічені.

## Як верифікувалися перенесення (для наступного виконавця)

Кожен спліт робився python-скриптом з assert-якорями на межі блоків, потім:
1. `gdparse` кожного файлу (обмеження: не розуміє багаторядкові рядкові
   літерали GDScript — `main.gd:745` дає хибний фейл і на HEAD, і зараз);
2. звірка нормалізованих тіл функцій (без пробілів/коментарів, зі знятими
   доданими префіксами) — 100% збіг;
3. звірка, що зовнішні точки входу (`grep` по всіх скриптах) покриті
   делегатами/фасадом.
